1. Executive Summary


NexusHire is a graph-first, AI-powered job portal that transforms hiring from a one-directional application pipeline into a transparent, bidirectional relationship network. Unlike LinkedIn, Naukri, or any existing ATS, NexusHire makes the compatibility graph visible to both sides — candidates see exactly why they matched or failed, recruiters see applicants as a live visual graph rather than an endless spreadsheet.

The platform is built for a 24-hour hackathon by a team of 4, using a 100% free (or credit-covered) tech stack accessible from India. It directly addresses all four PS3 deliverables plus all three brownie point requirements.

2. Problem Statement Analysis


2.1  Core Problems Identified
The modern hiring ecosystem has two root failures that compound each other:

Discovery Failure: Candidates cannot find jobs that genuinely match their profile. Search is keyword-based, not semantic. Relevant opportunities are invisible.
Trust Failure: Applications vanish into a black hole. 80% of candidates never hear back. Both sides have learned to expect nothing, so both sides stop investing in the process.

4. Solution Overview


NexusHire is built on one central architectural insight:

Core Insight
Every application is not a form submission — it is an edge in a live, bidirectional graph between a candidate node and a job node, with an explainable compatibility score as its weight. Both nodes can see the edge and understand why it exists.

This is what the PS means by 'Application Graph Engine' and 'connected opportunity graph.' No existing platform makes this graph visible to both sides. NexusHire does.

4.1  Key Differentiators
Bidirectional score visibility: Both candidate and recruiter see the same compatibility score with the same explanation. No opacity.
Graph as primary UI: The recruiter's default view is a D3.js force-directed graph, not a list. Spatial position and node size encode match quality instantly.
Rejection with direction: Candidates receive a specific skill gap analysis and a Groq-generated learning path on every rejection — not a form letter.
Real-time pipeline transparency: Candidates track application status like a package. Firestore — replaced by MongoDB Change Streams + Pusher — keeps both sides in sync instantly.
No black hole: The pipeline tracker shows current stage, days elapsed, and percentile rank among all applicants — at all times.

6. System Architecture


6.1  Request Flow — Candidate Applies to a Job
Candidate submits application from Next.js frontend
POST /api/applications API route is called
API fetches candidate embedding + job embedding from MongoDB
Cosine similarity computed in Atlas Vector Search (or server-side JS fallback)
Groq Llama 3.1 8B called with candidate profile JSON + job entity JSON — returns match score object with overall score, skill breakdown, and explainability text
Application document created in MongoDB with score, breakdown, and stage = 'applied'
Response sent to frontend immediately — candidate sees match score breakdown

6.2  Request Flow — Recruiter Moves Candidate to New Stage
Recruiter drags candidate card in Kanban board (or clicks in graph sidebar)
PATCH /api/applications/:id API route updates the stage field in MongoDB
MongoDB Change Stream detects the update
Server-side Pusher client triggers event on channel: job-{jobId}
Candidate's browser (subscribed to application-{applicationId} channel) receives the push
React state updates, pipeline tracker re-renders — zero page refresh

6.3  Semantic Search Flow
Candidate types natural language query: 'backend engineer with Node'
GET /api/jobs/search?q=... — API calls NVIDIA NIM (llama-nemotron-embed-1b-v2) to embed the query string into a 768-dim vector
MongoDB Atlas Vector Search $vectorSearch aggregation finds top-K closest job embeddings
Results sorted by vector similarity score, filtered by active status
Each result includes the candidate's precomputed match score for that job
Frontend renders ranked job cards with match percentage badges

jobs document structure

  _id: ObjectId,
  recruiterId: ObjectId,
  title: "Senior Backend Engineer",
  description: "Raw JD text from recruiter",
  requiredSkills: [
    { skill: "Node.js", weight: 3, type: "technical" },
    { skill: "MongoDB", weight: 2, type: "technical" },
    { skill: "System Design", weight: 3, type: "technical" },
    { skill: "Communication", weight: 1, type: "soft" }
  ],
  experienceRange: { min: 3, max: 6 },
  roleLevel: "senior",
  cultureTags: ["remote-first", "fast-paced"],
  salaryRange: { min: 1200000, max: 1800000, currency: "INR" },

  _id: ObjectId,
  candidateId: ObjectId,

13. Demo Script — The Pitch Playbook


Goal
The judges should leave thinking: 'That is what job portals should have been all along.' Three moments create this: (1) the graph reveal, (2) the real-time pipeline sync, (3) the live interview simulator.

13.1  Opening Hook (30 seconds)
Open with a single statistic: '80% of job applications are never seen by a human. Both candidates and recruiters have learned to ghost each other. We did not build a better job board. We built the thing job boards were always supposed to be — a transparent network where both sides of every application can finally see each other.'

13.2  Demo Sequence (3 minutes)
Recruiter posts a job — show the AI parsing the JD into a structured entity in real time. Point out: 'Not free text. A machine-readable object.'
Candidate uploads resume — Groq Llama 3.1 8B parses it into a structured profile in real time. Point out the skill tags, proficiency inference.
Candidate searches for 'backend engineer' — show semantic results appearing that do not contain those exact words. Point out: 'This is not keyword matching.'
Candidate applies — match score breakdown appears immediately. Per-skill checklist. Percentile rank. 'You are in the top 18% for this role.' No black hole.
Switch to recruiter screen — open the applicant graph. Pause here. Let the visual breathe. Nodes are flying in, sized by match. Ask: 'Can you tell who the best candidates are just by looking?' The answer is yes.
Click the largest node — CandidateInspector slides in. Radar chart. Skill checklist. 'Move to Shortlisted.' Click.
Switch back to candidate screen — the pipeline tracker updates live. Real-time. 'The black hole is gone.'
If time: open Interview Simulator. Let a judge volunteer. Groq Llama 3.3 70B asks a role-specific question live. Show the answer evaluation. Show the feedback report.
